## Overview
The `postw90.F90` file contains the main program `postw90`, which serves as the primary executable for post-processing tasks using Wannier90. After Maximally Localised Wannier Functions (MLWFs) have been generated by `wannier90.x`, `postw90.x` can be used to calculate a variety of material properties by interpolating quantities (like Hamiltonian, position, or spin operators) in the Wannier basis onto dense k-meshes or specific k-paths.

The program orchestrates calls to various specialized modules to perform these calculations, such as Berry phase related properties (AHC, orbital magnetization, Kubo conductivity, SHC), Boltzmann transport, density of states, band structure interpolation, and k-space slice plots.

## Key Components

### Program `postw90`
- **Description:** The main driver program for all post-processing functionalities.
- **Workflow:**
    1.  **Initialization:** Sets up MPI, parses command-line arguments (`seedname`, `dryrun`) using `io_commandline`.
    2.  **Parameter Reading (Root Node):**
        - Calls `w90_postw90_readwrite_read` to parse the `seedname.win` input file, loading parameters for various calculations (DOS, k-path, k-slice, Berry phase properties, BoltzWann, gyrotropic effects, generic interpolation).
        - Prints a summary of parameters using `w90_postw90_readwrite_write`.
        - If not an `effective_model` run (i.e., not reading $H(\mathbf{R})$ directly), it calls `kmesh_get` to determine k-point neighbor information from the original `wannier90.x` run's k-mesh.
    3.  **Parameter Distribution:** Broadcasts all necessary parameters to other MPI nodes using `pw90common_wanint_w90_wannier90_readwrite_dist` and `pw90common_wanint_data_dist`.
    4.  **Checkpoint Reading (Root Node):** If not an `effective_model`, reads the `seedname.chk` file using `w90_readwrite_read_chkpt` to obtain $U$ matrices, Wannier centers, spreads, etc.
    5.  **Data Distribution:** Distributes checkpoint data (or constructs necessary matrices like `v_matrix = u_matrix_opt * u_matrix`) to all nodes.
    6.  **k-point List for Interpolation:** If `wanint_kpoint_file` is true, reads a user-supplied k-point list from `kpoint.dat`.
    7.  **Common Setup:** Calls `pw90common_wanint_setup` for common initializations for interpolation tasks.
    8.  **Task Execution:** Based on flags set in `pw90_calcs` (populated from input file):
        - Calls `dos_main` if `pw90_calcs%dos` is true.
        - Calls `k_path` if `pw90_calcs%kpath` is true.
        - Calls `k_slice` if `pw90_calcs%kslice` is true.
        - Calls `spin_get_moment` if `pw90_calcs%spin_moment` is true.
        - Calls `berry_main` if `pw90_calcs%berry` is true.
        - Calls `geninterp_main` if `pw90_calcs%geninterp` is true.
        - Calls `boltzwann_main` if `pw90_calcs%boltzwann` is true.
        - Calls `gyrotropic_main` if `pw90_calcs%gyrotropic` is true.
    9.  **Finalization:** Prints total execution time and timing summary, then finalizes MPI.
- **Error Handling:** Uses a local `prterr` subroutine to print error messages and stop execution, ensuring MPI synchronization of errors.

### Local Error Subroutine `prterr`
- **Description:** A simplified version of the error printing routine found in `wannier_prog.F90`, adapted for `postw90.x`. It ensures error messages are handled correctly in an MPI environment.

## Important Variables/Constants
The `postw90` program uses many variables that are instances of derived types defined in `w90_types.F90`, `w90_wannier90_types.F90`, and `w90_postw90_types.F90`. Key variables managed at the program level include:
- `seedname` (character): Base name for files.
- `stdout`, `stderr` (integer): File units for output and error logs.
- `real_lattice(3,3)`, `recip_lattice(3,3)`: Lattice vectors.
- `kpt_latt(:, :)`: Coordinates of the original k-mesh from `wannier90.x`.
- `num_bands`, `num_wann`, `num_kpts`: Core dimensions.
- `u_matrix`, `u_matrix_opt`, `v_matrix`: Unitary transformation matrices.
- `eigval`: Eigenvalues from the original `wannier90.x` run.
- `HH_R`, `AA_R`, `BB_R`, `CC_R`, `SS_R`, etc.: Allocatable arrays to store real-space matrix elements of operators, populated by `w90_get_oper` and used by various calculation routines.
- Instances of control types: `pw90_calcs`, `berry`, `boltz`, `dos_data`, `kpath`, `kslice`, `gyrotropic`, `geninterp`, `pw90_spin`, `pw90_ham`, `postw90_oper`.
- `comm` (type `w90comm_type`): MPI communicator.
- `timer` (type `timer_list_type`): For timing code sections.
- `error` (type `w90_error_type`): For error handling.

## Usage Examples
`postw90.x` is a command-line executable.
```bash
postw90.x myseedname
```
This command will read settings from `myseedname.win` and the checkpoint file `myseedname.chk` (and other potential files like `myseedname.mmn`, `myseedname.eig` if not an `effective_model` run or if checkpoint is incomplete). It will then perform the calculations specified in `myseedname.win` (e.g., DOS, band structure, AHC). Output is written to `myseedname.wpout` (log) and various data files (e.g., `myseedname-dos.dat`, `myseedname-bands.dat`).

## Dependencies and Interactions

- **Internal Dependencies:**
    - `w90_berry`: For Berry phase related properties.
    - `w90_boltzwann`: For Boltzmann transport calculations.
    - `w90_comms`: For MPI communication.
    - `w90_constants`: For physical constants and precision.
    - `w90_dos`: For Density of States calculations.
    - `w90_error`, `w90_error_base`: For error handling.
    - `w90_geninterp`: For generic k-point interpolation.
    - `w90_gyrotropic`: For gyrotropic effects calculations.
    - `w90_io`: For command-line parsing, timing, version info.
    - `w90_kmesh`: For k-mesh utilities (called if not `effective_model`).
    - `w90_kpath`: For calculations along k-space paths.
    - `w90_kslice`: For calculations on k-space slices.
    - `w90_postw90_common`: For shared setup and data distribution routines specific to `postw90.x`.
    - `w90_postw90_readwrite`: For parsing `postw90.x`-specific input parameters.
    - `w90_postw90_types`: For `postw90.x`-specific derived data types.
    - `w90_readwrite`: For reading checkpoint files and common input parameters.
    - `w90_spin`: For spin-related calculations.
    - `w90_types`: For shared derived data types.
    - (Implicitly, through called modules) `w90_get_oper`, `w90_wan_ham`, `w90_utility`.

- **External Libraries:**
    - **MPI:** If compiled with MPI support.

- **Interactions:**
    - Reads `seedname.win` for controlling parameters.
    - Reads `seedname.chk` for MLWF data ($U$ matrices, centers, spreads, $M$ matrices if stored).
    - May read other files like `seedname.mmn`, `seedname.eig`, `seedname.spn`, `seedname.uHu`, etc., if these are needed by `w90_get_oper` to construct real-space matrix elements and `effective_model = .false.`.
    - Calls various computational modules to perform post-processing tasks.
    - Writes results to `seedname.wpout` (log file) and numerous specialized data and script files.
